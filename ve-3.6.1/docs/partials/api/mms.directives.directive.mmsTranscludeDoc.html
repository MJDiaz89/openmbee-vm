mmsTranscludeDoc
directive in module mms.directives

Description

Given an element id, puts in the element's documentation binding, if there's a parent mmsView directive, will notify parent view of transclusion on init and doc change, and on click. Nested transclusions inside the documentation will also be registered.

Example

    <mms-transclude-doc mms-element-id="element_id"></mms-transclude-doc>

Dependencies

  • ElementService
  • UtilsService
  • ViewService
  • UxService
  • Utils
  • $compile
  • $templateCache
  • growl
  • _

Usage

as element:
<mms-transclude-doc
       mms-element-id="{string}"
       mms-project-id="{string}"
       mms-ref-id="{string=master}"
       mms-commit-id="{string=latest}"
       mms-watch-id="{bool}"
       non-editable="{boolean=false}">
</mms-transclude-doc>

Parameters

ParamTypeDetails
mmsElementIdstring

The id of the view

mmsProjectIdstring

The project id for the view

mmsRefIdstring=master

Reference to use, defaults to master

mmsCommitIdstring=latest

Commit ID, default is latest

mmsWatchIdbool

set to true to not destroy element ID watcher

nonEditableboolean=false

can edit inline or not
