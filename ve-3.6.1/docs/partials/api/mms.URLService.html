URLService
service in module mms

Description

This utility service gives back url paths for use in other services in communicating with the server, arguments like projectId, refId, commitId are expected to be strings and not null or undefined. This service is usually called by higher level services and should rarely be used directly by applications.

To configure the base url of the mms server, you can use the URLServiceProvider in your application module's config. By default, the baseUrl is '/alfresco/service' which assumes your application is hosted on the same machine as the mms and ve. 

        angular.module('myApp', ['mms'])
        .config(function(URLServiceProvider) {
            URLServiceProvider.setBaseUrl('https://url/alfresco/service');
        });
(You may run into problems like cross origin security policy that prevents it from actually getting the resources from a different server, solution TBD)
